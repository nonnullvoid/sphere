How it works

Your AI reads your files.
Every session starts oriented.

Sphere is a folder of markdown files. A single CLAUDE.md at the root tells your AI to read them at startup — your profile, your principles, how you like to work. The AI arrives already knowing the context. You pick up where you left off.

01

Write your context once

Fill in core/user/ — your profile, your principles, how you like to work. The onboard skill walks you through it in conversation so you're not staring at blank files.

02

CLAUDE.md loads it at startup

A single file at the root instructs your AI to read your sphere when you open the directory. No configuration. No plugins. It just reads the files.

03

The difference is immediate

Your AI doesn't ask who you are. It doesn't need to be caught up. It starts from your context, not from zero — and it stays that way every session.

Without Sphere

Architecture

A sphere has structure.

Three layers. Everything in the right place.

core/
layers/
surface/

Inside every sphere

⬤
core/
The fixed center. Your identity (user/) and the AI persona (ai/persona.md). Loaded at every session start — this is what makes the AI know you.
◎
layers/
Optional domain extensions — philosophy, wellness, financial, projects. Loaded on-demand when the work requires them. Install only what fits your life.
◯
surface/
Artifacts and outputs — documents, research, drafts. What the sphere produces, kept separate from what it is.
Get started

Clone a template. Fill it in.

Two starting points. Start with the blank template if you want to build from scratch. Start with a pre-structured template if you want more scaffolding for a specific context.

Blank template

The minimal starting point — structure only. The onboard skill fills it in through conversation so you're not staring at empty files.
  • Core structure, no pre-filled content
  • Onboard skill — guided conversational setup
  • Reflect skill — periodic sphere evolution
Recommended

Pre-structured templates

Two templates with domain-specific scaffolding — one for life context, one for professional work. More structure to guide what goes where.
  • LifeSphere — identity, roles, values, reflection
  • WorkSphere — domain, projects, professional context
  • Both base skills included
  • Fork and replicate skills for sharing and cloning

Clone and set up

# Clone the repo
git clone https://github.com/rutvij47/sphere.git

# Copy the template you want
cp -r sphere/templates/seed ~/my-sphere
# or: cp -r sphere/templates/life-sphere ~/my-sphere
# or: cp -r sphere/templates/work-sphere ~/my-work-sphere

# Open in Claude Code — sphere loads automatically
cd ~/my-sphere && claude
